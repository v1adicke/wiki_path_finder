# Wiki Path Finder

Реализация популярной игры - поиска кратчайшего пути между двумя статьями русской Википедии, представленная telegram-ботом на базе aiogram: @wikipathfinder_bot

---

### Структура проекта

```
wiki_path_finder/
├── .env
├── requirements.txt
├── README.md
├── telegram_bot/
│   ├── main.py # приложение бота
│   ├── handlers.py # обработчики бота
│   └── title_validator.py # валидация пользовательского ввода
└── search/
    ├── api_client.py # клиент для обращения к API
    ├── path_finder.py # алгоритм поиска
    └── result.py # форматирование результата
```

### Требования

- Python 3.10+
- aiogram>=3.0
- python-dotenv
- aiohttp

### Установка и запуск

- Склонируйте репозиторий локально
- Создайте и активируйте виртуальное окружение
- Установите зависимости командой: pip install -r requirements.txt
- Создайте файл .env в корне проекта с содержимым: BOT_TOKEN=ваш_токен_бота
- Запустите бота из корня проекта командой: python -m telegram_bot.main

### Команды бота

- /start - запуск: введите начальную статью, затем после успешной проверки — конечную статью для запуска поиска.
- /help - краткая справка о боте.

### Технические особенности

- Поиск пути реализован двунаправленным обходом графа статей: фронт от начальной статьи расширяется по исходящим ссылкам, фронт от конечной — по обратным ссылкам, что уменьшает глубину поиска до встречи.
- Получение ссылок и обратных ссылок реализовано асинхронно через aiohttp с обработкой ошибок.
- Все модули проекта изолированы и независимы.
