# Wiki Path Finder

Реализация популярной игры - поиска кратчайшего пути между двумя статьями русской Википедии, представленная telegram-ботом на базе aiogram: @wikipathfinder_bot

### Команды бота

- /start - запуск: введите начальную статью, затем после успешной проверки — конечную статью для запуска поиска
- /help - краткая справка о боте

### Технические особенности

- Поиск пути реализован двунаправленным обходом графа статей: фронт от начальной статьи расширяется по исходящим ссылкам, фронт от конечной — по обратным ссылкам, что уменьшает глубину поиска до встречи
- Поиск выполняется в два этапа: быстрый эвристический проход (ограничение ветвления по релевантным статьям) и точный fallback-проход без ограничений в оставшееся время
- Получение ссылок и обратных ссылок реализовано асинхронно через aiohttp с обработкой ошибок
- В API-клиент добавлены LRU-кеши и дедупликация одинаковых запросов что уменьшает число повторных обращений к Wikipedia API
- Все модули проекта изолированы и независимы

### Тесты производительности

Для стресс-тестирования поиска добавлен отдельный сервис бенчмаркинга.

Что делает сервис:
- Генерирует большой набор запросов разной сложности (`easy`, `medium`, `hard`, `very_hard`)
- Добавляет синтетические сложные словосочетания (в том числе несуществующие в Википедии)
- Массово прогоняет поиск с контролем тайм-лимита и параллелизма
- Считает метрики (успех, время, статусы, длина путей)
- Строит PNG-дашборд на белом фоне с логотипом в центре и отдельными панелями без наложения графиков
- Сохраняет JSON-отчёт для последующего анализа

Команда запуска:

```bash
python -m benchmarking --total-cases 240 --time-limit 40 --concurrency 8 --logo assets/logo.png --out-image reports/dashboard.png --out-json reports/metrics.json
```

Параметры:
- `--total-cases`: количество тестовых запросов
- `--time-limit`: тайм-лимит на один поиск (сек)
- `--concurrency`: сколько запросов выполнять параллельно
- `--logo`: путь к вашему логотипу (положите файл в `assets/logo.png`)
- `--out-image`: путь к PNG-дашборду
- `--out-json`: путь к JSON с сырыми метриками

Артефакты:
- `reports/dashboard.png` - визуальный дашборд
- `reports/metrics.json` - подробная статистика по каждому кейсу

Рекомендация для тяжелого стресс-теста:
- `--total-cases 500 --time-limit 60 --concurrency 10`
